### 核心原则（必须严格遵守，按优先级排序）

1. **翻译准确性第一位**  
   - 所有 Locust 专业术语必须保持中英文对照，在一个页面正文首次出现时使用：中文名(英文原文) 格式，例如：
     - 分布式模式(Distributed Mode)
     - 用户(User)
     - 任务集(TaskSet)
     - 形状(Shape)
     - 孵化速率(Hatch Rate) → 注意：统一翻译为「孵化速率」，不再使用「孵化率」
   - 绝不能为了「通顺」而牺牲准确性。若直译拗口，必须在括号内保留原文，或在后续补充说明。
   - 所有代码、命令行、配置文件中的英文一律保持原样，不翻译。

2. **性能与工程最优化**
   - 文档必须使用 Markdown + MDX，优先使用最新特性。
   - 所有图片使用 WebP 或 AVIF 格式，建议 < 100KB，优先使用 SVG。
   - 启用预加载、预取、按需加载，所有代码块必须支持一键复制。
   - 侧边栏、导航必须实现客户端持久化展开状态。
   - 所有页面必须在 Lighthouse 中达到 Performance ≥ 98，Accessibility 100，Best Practices 100，SEO 100。

3. **设计高级感与用户体验极致**
   - 配色方案：浅色模式为主，采用 Locust 官方绿(#35563a)作为主色，辅以极简留白设计。
   - 字体：中文优先使用「小米兰亭 Pro」或「HarmonyOS Sans」，英文使用「Geist Mono」「JetBrains Mono」。
   - 所有表格必须响应式，手机端自动堆叠。
   - 针对性考虑中文开发者的使用体验。
   - 每个代码块右侧必须有关联的「运行」「复制」三个图标。
   - 所有页面顶部必须有「最后更新时间」和「编辑此页」（指向 GitHub）。

4. **中文技术写作规范（强制）**
   - 语气：专业但不严肃，友好且鼓励尝试，参考 Python 官方中文文档的语气。
   - 标点：全角中文标点，英文及代码内使用半角。
   - 人称：统一使用「你」而非「您」或「我们」，具体以英文原文为准。
   - 专有名词大小写保持一致，例如：Locust、Web UI、Master、Worker（首字母大写）。
   - 动词统一，具体以英文原文为准：
     - 「运行」而不是「执行」「启动」
     - 「停止」而不是「终止」
     - 「孵化」而不是「生成」「创建」用户

5. **文件与目录结构规范**
   - 所有文件使用 kebab-case。
   - 每篇文档必须包含 frontmatter，包含 title、description、sidebar_position、lastUpdated 等。
